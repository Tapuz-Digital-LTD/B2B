# עמודי קמפיין נבנים מספריית סקשנים משותפת + קבצי תוכן בריפו (בלי CMS)

האתר מורכב מעמוד בית גנרי ועמודי קמפיין עונתיים (פסח, ראש השנה...) שחולקים את אותו מבנה סקשנים כשרק התוכן משתנה; קמפיין רשאי לחרוג לעיצוב ייחודי. החלטנו שהתוכן מנוהל בקבצי תוכן טיפוסיים בריפו (`src/content/<page>.ts`) שמוזרמים לסקשנים כפרופס — לא ב-CMS — כי קמפיינים מתחלפים מעט פעמים בשנה, כל החלפה ממילא כוללת עבודת עיצוב/נכסים שעוברת דרך מפתח, וקובץ תוכן נותן את רוב הגמישות בשבריר מהמורכבות. אם בעתיד צוות שיווק יזדקק לעריכה עצמאית, מחברים CMS לשכבת הטקסטים בלבד מבלי לשנות את הסקשנים.

## Considered Options

- CMS חיצוני (Sanity/Contentful) — נדחה: תקורת תשתית ותחזוקה בלי צורך אמיתי בשלב הזה.
- טקסטים קשיחים בתוך הקומפוננטות (המצב ההתחלתי) — נדחה: כל קמפיין היה מחייב שכפול עמוד שלם.
